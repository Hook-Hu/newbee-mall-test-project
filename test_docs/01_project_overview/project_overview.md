# newbee-mall-plus 项目功能梳理

## 一、项目基本信息

| 项目 | 内容 |
|---|:--|
| 项目名称 | newbee-mall-plus 商城测试项目 |
| 项目类型 | B2C 商城系统 |
| 项目来源 | 开源商城系统 |
| 项目定位 | 软件测试求职项目 / 测试能力展示项目 |
| 测试目标 | 基于真实商城业务场景，完成从功能测试、接口测试、自动化测试到性能测试的完整测试实践 |

本项目基于 `Spring Boot` 商城系统搭建测试环境，围绕用户登录、商品浏览、购物车、订单提交、优惠券、秒杀、后台管理等核心业务模块，开展功能测试、接口测试、异常场景测试、自动化测试和性能测试实践。

## 四、系统组成

本项目主要包含两个部分：前台商城系统和后台管理系统。

### 1. 前台商城系统

前台商城主要面向普通用户，包含用户注册登录、商品浏览、商品搜索、购物车、订单提交、优惠券、秒杀、支付、个人中心等功能。

### 2. 后台管理系统

后台管理系统主要面向管理员，包含管理员登录、商品管理、分类管理、订单管理、用户管理、轮播图管理等功能。

---

## 五、测试范围

本轮测试主要覆盖以下内容：

- 前台商城核心购买流程；
- 后台商品、订单、分类等基础管理流程；
- 图片资源、本地上传目录和静态资源访问；
- 接口参数校验、业务状态校验和异常场景；
- 登录、购物车、订单、秒杀等高风险业务链路；
- 秒杀场景下的库存扣减、重复下单和并发控制；
- 前台操作与后台数据展示的一致性。

---

## 六、不在本轮测试范围

以下内容不作为本轮重点测试范围：

- 第三方支付真实交易；
- 线上短信、邮件等外部服务；
- 多端兼容性深度测试；
- 生产级安全渗透测试；
- 大规模线上稳定性测试。

## 七、核心业务模块与测试关注点

| 模块 | 主要功能 | 测试关注点 |
|---|---|---|
| 首页模块 | 轮播图、商品分类、推荐商品、页面资源加载 | 页面是否正常加载，轮播图和商品数据是否展示，图片资源是否可访问 |
| 用户模块 | 注册、登录、退出、个人信息、收货地址 | 正确登录、错误登录、登录状态保持、未登录拦截、收货地址增删改查 |
| 商品模块 | 商品列表、商品详情、分类筛选、商品搜索 | 商品信息展示、价格库存展示、分类筛选、关键词搜索、分页和排序 |
| 购物车模块 | 加入购物车、修改数量、删除商品、勾选商品、结算 | 未登录拦截、数量修改、金额计算、删除商品、勾选结算 |
| 订单模块 | 提交订单、订单详情、订单状态流转、取消订单 | 下单成功、库存校验、金额计算、状态流转、取消订单后库存变化 |
| 优惠券模块 | 优惠券展示、领取、使用、状态变化 | 重复领取限制、使用条件校验、优惠金额计算、使用后状态更新 |
| 秒杀模块 | 秒杀商品展示、秒杀下单、库存扣减、重复下单控制 | 活动时间校验、库存不足、超卖问题、重复秒杀、Redis 与 MySQL 数据一致性 |
| 后台管理模块 | 管理员登录、商品管理、分类管理、订单管理、用户管理 | 管理员登录、商品增删改查、订单状态查看、前后台数据一致性 |
