# 接口测试

## 测试目标

- 验证接口可用性、参数校验、鉴权控制和业务数据一致性。
- 为自动化接口回归准备接口清单和断言依据。

## 接口测试范围

| 模块 | 接口类型 | 关注点 |
| --- | --- | --- |
| 用户 | 登录、退出、用户信息 | 登录状态、错误提示、会话保持 |
| 商品 | 列表、详情、搜索 | 参数校验、分页、商品状态 |
| 购物车 | 新增、修改、删除、查询 | 数量限制、金额计算、登录校验 |
| 订单 | 创建、查询、取消 | 库存扣减、状态流转、重复提交 |
| 后台 | 商品、分类、订单管理 | 管理员权限、字段校验、数据落库 |

## 通用断言

- 响应状态码符合预期。
- 返回体结构稳定。
- 业务状态码和提示信息正确。
- 数据库数据新增、修改或删除符合预期。
- 未登录访问受保护接口时应被拦截。
- 越权访问后台接口时应被拒绝。

## 测试数据准备

| 数据类型 | 准备方式 | 备注 |
| --- | --- | --- |
| 普通用户 | 数据库预置或页面注册 | 用于前台流程 |
| 管理员 | 初始化数据 | 用于后台接口 |
| 商品数据 | 初始化脚本或后台新增 | 覆盖上架、下架、库存不足 |
| 订单数据 | 前台下单生成 | 覆盖不同订单状态 |
