# 注册接口测试记录

## 一、测试环境

| 项目 | 内容 |
|---|---|
| 测试工具 | Apifox |
| 测试环境 | 本地环境 |
| base_url | http://localhost:28079 |
| 测试模块 | 用户注册 |
| 前置接口 | GET /common/mall/kaptcha |
| 测试接口 | POST /register |

## 二、接口说明

注册接口用于商城前台用户注册，用户需要提交登录名、密码和图片验证码。验证码由 `/common/mall/kaptcha` 接口生成，并依赖同一会话 Cookie 进行校验。

## 三、请求参数

| 参数名 | 是否必填 | 说明 |
|---|---|---|
| loginName | 是 | 注册账号 |
| password | 是 | 注册密码 |
| verifyCode | 是 | 图片验证码 |

## 四、测试用例执行记录

| 编号 | 测试场景 | 请求参数 | 预期结果          | 实际结果 | 是否通过 | 断言 |
|---|---|---|---------------|---|---|---|
| API-REGISTER-001 | 正常注册 | 新账号、正确密码、正确验证码 | 注册成功          | resultCode=200，message=SUCCESS | 通过 | $.resultCode=200；$.message=SUCCESS |
| API-REGISTER-002 | 验证码错误 | 新账号、正确密码、错误验证码 | 注册失败，提示验证码错误！ | resultCode=500，message 包含验证码错误 | 通过 | $.resultCode=500；$.message 包含验证码错误 |
| API-REGISTER-003 | 重复账号注册 | 已注册账号、正确密码、正确验证码 | 注册失败，提示用户名已存在！ | resultCode=500，message 包含已存在/已注册 | 通过 | $.resultCode=500；$.message 包含对应错误信息 |

## 五、测试结论

本轮注册接口测试覆盖了正常注册、验证码错误、重复账号注册三个核心场景。测试结果表明，注册接口能够正确处理正常注册请求，并能对验证码错误、重复账号等异常场景返回对应的业务失败结果。
