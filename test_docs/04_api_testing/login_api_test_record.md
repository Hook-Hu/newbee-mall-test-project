# 登录接口测试记录

## 一、测试环境

| 项目 | 内容 |
|---|---|
| 测试工具 | Apifox |
| 测试环境 | 本地环境 |
| base_url | http://localhost:28079 |
| 测试模块 | 用户登录 |
| 前置接口 | GET /common/mall/kaptcha |
| 测试接口 | POST /login |

## 二、接口说明

登录接口用于商城前台用户登录，用户需要提交登录名、密码和图片验证码。验证码由 `/common/mall/kaptcha` 接口生成，并依赖同一会话 Cookie 进行校验。

## 三、请求参数

| 参数名 | 是否必填 | 说明 |
|---|---|---|
| loginName | 是 | 用户登录名 |
| password | 是 | 用户密码 |
| verifyCode | 是 | 图片验证码 |

## 四、测试用例执行记录

| 编号 | 测试场景 | 请求参数 | 预期结果 | 实际结果                           | 是否通过 | 断言 |
|---|---|---|---|--------------------------------|---|---|
| API-LOGIN-001 | 正常登录 | 正确账号、正确密码、正确验证码 | 登录成功 | resultCode=200，message=SUCCESS | 通过 | $.resultCode=200；$.message=SUCCESS |
| API-LOGIN-002 | 验证码错误 | 正确账号、正确密码、错误验证码 | 登录失败，提示验证码错误 | resultCode=500，message=验证码错误！  | 通过 | $.resultCode=500；$.message 包含 验证码错误 |
| API-LOGIN-003 | 密码错误 | 正确账号、错误密码、正确验证码 | 登录失败，提示登录失败 | resultCode=500，message=登录失败!   | 通过 | $.resultCode=500；$.message 包含 登录失败 |

## 五、测试结论

本轮登录接口测试覆盖了正常登录、验证码错误、密码错误三个核心场景。  
测试结果表明，登录接口能够正确处理正常登录请求，并能对验证码错误和密码错误场景返回对应的业务失败结果。

需要注意的是，该项目登录接口依赖图片验证码和 Session，因此在接口测试时需要先请求验证码接口，并保持同一会话 Cookie。
