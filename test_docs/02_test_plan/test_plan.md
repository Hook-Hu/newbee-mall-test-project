# 测试计划

## 测试目标

- 验证商城核心业务流程可以正常运行。
- 发现影响用户下单、商品展示、后台管理的关键缺陷。
- 建立可复用的测试用例、接口测试和自动化测试资料。
- 为后续面试复盘准备完整项目材料。

## 测试阶段

| 阶段 | 目标 | 输出物 |
| --- | --- | --- |
| 环境验证 | 确认项目、数据库、缓存、图片资源正常 | 环境检查记录 |
| 冒烟测试 | 快速验证主流程可用 | 冒烟测试结论 |
| 功能测试 | 覆盖前后台核心功能 | 功能测试用例和缺陷 |
| 接口测试 | 验证接口状态码、参数、返回体和业务数据 | 接口测试记录 |
| 自动化测试 | 固化高频回归路径 | 自动化脚本和执行报告 |
| 性能测试 | 验证关键链路承压能力 | 性能测试报告 |
| 回归测试 | 验证缺陷修复和主流程稳定性 | 回归测试报告 |

## 准入标准

- 项目可以正常启动。
- `MySQL` 和 `Redis` 服务可用。
- 基础测试数据已导入。
- 首页、商品详情、后台登录可以访问。

## 准出标准

- 严重和阻塞级缺陷已修复并回归通过。
- 核心购买链路测试通过。
- 后台核心管理链路测试通过。
- 接口测试主要场景通过。
- 测试报告已输出。

## 风险点

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 图片路径异常 | 商品详情、首页展示异常 | 检查上传目录和数据库图片路径 |
| 数据库脚本不完整 | 项目启动或业务流程异常 | 保留初始化脚本和补充脚本 |
| 秒杀并发问题 | 库存超卖或请求失败 | 增加并发测试和库存校验 |
| 测试数据污染 | 用例结果不稳定 | 准备数据重置方案 |
