# 自动化测试

## 自动化目标

- 固化商城核心回归流程。
- 降低重复手工回归成本。
- 为测试项目展示提供可运行脚本和报告。

## 自动化范围

| 类型 | 场景 |
| --- | --- |
| 页面自动化 | 首页访问、登录、商品搜索、加入购物车、提交订单 |
| 接口自动化 | 登录接口、商品接口、购物车接口、订单接口 |
| 冒烟自动化 | 项目启动后快速检查首页、后台、关键接口是否可用 |

## 暂不自动化的场景

- 验证码强依赖场景。
- 频繁变化的后台页面样式细节。
- 需要人工判断展示效果的视觉问题。
- 第三方支付真实链路。

## 脚本规划

| 目录 | 说明 |
| --- | --- |
| `smoke` | 冒烟测试脚本 |
| `web` | 页面自动化脚本 |
| `api` | 接口自动化脚本 |
| `data` | 测试数据 |
| `reports` | 测试报告 |
