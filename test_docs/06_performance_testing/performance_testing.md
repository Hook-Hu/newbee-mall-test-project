# 性能测试

## 测试目标

- 验证商城关键接口在并发访问下的响应能力。
- 发现数据库、缓存、接口逻辑中的性能瓶颈。
- 重点关注商品详情、搜索、加购物车、下单和秒杀场景。

## 性能场景

| 场景 | 说明 | 关注指标 |
| --- | --- | --- |
| 首页访问 | 多用户同时访问首页 | 响应时间、错误率 |
| 商品详情 | 高频查看商品详情 | 响应时间、数据库查询 |
| 商品搜索 | 多用户按关键字搜索 | 响应时间、分页查询 |
| 加入购物车 | 登录用户并发加入购物车 | 成功率、数据一致性 |
| 提交订单 | 多用户同时提交订单 | 成功率、库存扣减 |
| 秒杀下单 | 高并发抢购秒杀商品 | 超卖、接口错误率 |

## 指标定义

| 指标 | 说明 |
| --- | --- |
| 平均响应时间 | 所有请求响应时间平均值 |
| 百分位响应时间 | `P90`、`P95`、`P99` 响应时间 |
| 吞吐量 | 单位时间内处理请求数 |
| 错误率 | 失败请求占总请求比例 |
| 并发用户数 | 同时发起请求的用户数量 |
| 资源使用率 | `CPU`、内存、数据库连接、缓存连接 |

## 测试数据

| 数据 | 要求 |
| --- | --- |
| 用户 | 至少准备多组可登录用户 |
| 商品 | 准备普通商品、库存不足商品、秒杀商品 |
| 地址 | 每个用户至少存在一个收货地址 |
| 订单 | 测试前清理干扰数据 |
